<objective>
Guarantee that job log messages always persist to the database even when an exception — including BaseException subclasses like asyncio.CancelledError — aborts the job mid-run.

Purpose: The current runner.py calls _flush_log() in the try branch (happy path) and in the except Exception branch. A BaseException subclass (e.g., asyncio.CancelledError, KeyboardInterrupt) bypasses the except Exception handler entirely, so _flush_log() never runs and log lines are silently lost. Moving _flush_log() to a finally block closes this gap.

Output: runner.py with _flush_log() in a finally block (removed from try and except), plus a regression test that fails with the current code and passes after the fix.
</objective>
