<objective>
Cause Notion pre-check API errors to halt the job immediately rather than silently continuing.

Purpose: The current inner try/except around `mark_notion_duplicates()` swallows the error as a `[warn]` log line and lets the job continue, risking duplicate Notion pushes. Removing it lets any exception fall through to the existing outer handler, which already sets `job.status = "failed"` and flushes the log.

Output: One modified file (`runner.py`, inner try/except removed) and one new test function (`test_notion_precheck_error_fails_job` in `test_runner.py`). No new dependencies.
